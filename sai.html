1 • Smart Energy Grid
Adjacency List · Priority Queue
Dijkstra · Kruskal · Knapsack DP

Aarunya’s Smart Energy Grid links solar rooftops, wind farms and substations into a unified network that must move electricity from where it is generated to where it is needed with minimal loss. Sensor streams describe generation, demand and line limits, and the grid is modelled as a weighted graph so that Dijkstra can route energy along low‑loss paths between critical nodes like hospitals and industrial clusters. Kruskal’s algorithm designs new transmission links as a minimum‑spanning tree to minimise construction cost, while a knapsack-style DP distributes limited battery and renewable capacity across time slots to flatten peaks.

Aligned SDGs: SDG 7 – Affordable and Clean Energy, SDG 11 – Sustainable Cities and Communities

Data Structures: vector<Node>, vector<vector<pair<int,double>>>, priority_queue — Algorithms: Dijkstra, Kruskal MST, 0/1 Knapsack DP
Loads → Demand Profile → Knapsack DP → Time‑slot Allocation Grid Topology → Adjacency List → Dijkstra → Low‑loss Routes New Lines → Edge List → Kruskal + Union–Find → Minimum‑Cost Expansion

2 • AI Healthcare Prediction Hub
Decision Tree · Hash Tables
DP Outbreak Model · KMP

The healthcare hub ingests hospital visit logs, lab reports and ward‑level demographics to flag potential disease outbreaks before they become visible on the ground. Categorical features such as symptoms, age bands and locality are binned into arrays and hash tables, and a decision‑tree style classifier labels each ward as low, medium or high risk. A dynamic‑programming based compartment model then projects how many cases can emerge over the next few days, while KMP-style pattern matching scans health notes for dangerous keyword combinations that merit manual review.

Aligned SDGs: SDG 3 – Good Health and Well‑being, SDG 11 – Sustainable Cities and Communities

Data Structures: vector<Record>, unordered_map<string,int> — Algorithms: Decision‑tree classification, DP epidemic projection, KMP string matching
Ward Data → Hash Tables → Feature Vectors → Decision Tree → Risk Labels Risk Timeline → DP Model → Outbreak Curve Notes Text → KMP → Alert Phrases

3 • Smart Mobility & Traffic Control Hub
Graph · Adjacency Matrix
Dijkstra · Floyd–Warshall · Segment Tree

The traffic control centre receives live speeds from road sensors and must keep vehicles, buses and emergency services moving through hundreds of junctions. A dense adjacency matrix between key hubs feeds a Floyd–Warshall run that precomputes all‑pairs travel times for planning dashboards, while per‑request routing for cars and ambulances uses Dijkstra on a more detailed adjacency‑list graph. Segment trees maintain rolling traffic densities for corridors so that congestion alerts and signal timing changes can be triggered with fast range queries and updates.

Aligned SDGs: SDG 11 – Sustainable Cities and Communities, SDG 9 – Industry, Innovation and Infrastructure

Data Structures: adjacency list, adjacency matrix, segment tree — Algorithms: Dijkstra, Floyd–Warshall
Sensor Speeds → Graph Weights → Dijkstra → Live Shortest Routes Hub Matrix → Floyd–Warshall → All‑Pairs Travel Table Zone Flows → Segment Tree → Range Congestion Queries

4 • Satellite–IoT Disaster Monitoring Hub
Grid · Graph
BFS / DFS · DP Forecast · Huffman

The disaster hub fuses rainfall gauges, river‑level sensors and satellite pixels to understand how floods or landslides might spread across Aarunya. The terrain is discretised into a grid where BFS or DFS steps outward from a breach point to mark which cells will be affected next. A dynamic‑programming layer estimates hazard intensity over time by accumulating upstream inflows, and Huffman coding compresses large sensor snapshots before archiving so that historical simulations remain storage‑efficient.

Aligned SDGs: SDG 13 – Climate Action, SDG 11 – Sustainable Cities and Communities

Data Structures: 2D grid, adjacency list — Algorithms: BFS / DFS, DP hazard forecasting, Huffman coding
Sensors + DEM → Grid → BFS/DFS → Reachable Flood Cells Cell History → DP → Intensity Timeline Snapshots → Huffman Encoder → Compressed Archive

5 • Cybersecurity Intelligence Center
Hash Tables · Graph
Pattern Matching · Attack‑Path Search

Aarunya’s cybersecurity centre continuously scans network traffic from databases, cameras and IoT devices that support city operations. Hash‑based authentication tables validate device keys quickly, while KMP or Boyer–Moore pattern matching engines compare traffic payloads against known malware signatures. A graph of hosts and connections is used to trace likely attack paths so that security teams can see how a breach in one subsystem might pivot to others and can isolate critical nodes in time.

Aligned SDGs: SDG 9 – Industry, Innovation and Infrastructure, SDG 16 – Peace, Justice and Strong Institutions

Data Structures: unordered_map<string,string>, adjacency list — Algorithms: KMP / Boyer–Moore, graph traversal, hashing
Login / Device ID → Hash Table → Valid / Reject Packet Stream → Pattern Matcher → Threat Alerts Host Graph → Search → Attack Path Visualisation

6 • Autonomous Shuttle Pods
Graph · Grid
A* / Dijkstra · Priority Queue

Electric shuttle pods in Aarunya provide last‑mile connectivity between metro stations, offices and residential clusters. Each pod treats the street map as a weighted graph and uses A* or Dijkstra to plan safe, near‑optimal routes around blocked roads and dynamic obstacles. A global scheduler maintains a priority queue of pending ride requests, assigning pods based on proximity and waiting time so that average passenger delay stays low even during peak hours.

Aligned SDGs: SDG 11 – Sustainable Cities and Communities, SDG 7 – Affordable and Clean Energy

Data Structures: adjacency list, priority_queue — Algorithms: A* search, Dijkstra
Requests → Priority Queue → Next Passenger City Graph → A* / Dijkstra → Pod Route Live Obstacles → Cost Updates → Re‑plan if Needed

7 • Neural Urban Simulation Lab
AVL / Balanced Trees
DP Simulation · TSP

The simulation lab runs what‑if experiments for Aarunya’s planners, from new bus corridors to relocating waste‑processing plants. Historical metrics are indexed in balanced trees so that queries over any time window are efficient. Dynamic‑programming based models approximate how traffic, pollution and service access evolve over simulated years, and travelling‑salesperson style solvers optimise inspection or maintenance tours so that ground teams can validate assumptions with minimal travel.

Aligned SDGs: SDG 11 – Sustainable Cities and Communities, SDG 9 – Industry, Innovation and Infrastructure

Data Structures: AVL / ordered maps — Algorithms: DP‑based simulation, TSP heuristics, regression forecasting
Historical Data → Balanced Trees → Fast Range Queries Scenario Inputs → DP Engine → Multi‑year Projections Service Points → TSP Heuristic → Efficient Field Route

8 • Smart Lighting & Safety Network
Segment Tree · Graph
Greedy Energy Policy · BFS

Aarunya’s street‑lighting system links thousands of solar‑powered poles with motion and ambient‑light sensors. A segment tree keeps aggregated energy use for neighbourhoods so that demand caps and savings can be tracked quickly over ranges of streets. A greedy controller dims or brightens lights based on time of night and pedestrian activity, while BFS on a walkway graph ensures that there is always at least one safe, well‑lit path between key points such as bus stops and hospitals.

Aligned SDGs: SDG 7 – Affordable and Clean Energy, SDG 11 – Sustainable Cities and Communities

Data Structures: segment tree, adjacency list — Algorithms: greedy scheduling, BFS connectivity
Pole Sensors → Segment Tree → Zone Energy Stats Policy Rules → Greedy Controller → Dim / Brighten Commands Walkway Graph → BFS → Safe Night Routes

9 • Data Commerce & Analytics Hub
Hash Index · Trie
Sort · Rabin–Karp

The data hub exposes Aarunya’s curated datasets—traffic, air quality, utilities and more—to startups and researchers. Metadata about each dataset is indexed using tries to support instant auto‑completion when users type keywords, while hash tables map IDs to file locations for near constant‑time retrieval. Large logs are preprocessed with efficient sorting and Rabin–Karp style scanning so that abuse patterns and popular access paths can be analysed without re‑reading every character on each query.

Aligned SDGs: SDG 9 – Industry, Innovation and Infrastructure, SDG 17 – Partnerships for the Goals

Data Structures: Trie, unordered_map, vector — Algorithms: Merge / Quick Sort, Rabin–Karp
Dataset Tags → Trie → Search Suggestions ID → Hash Index → Dataset Location Access Logs → Sort + Rabin–Karp → Usage / Abuse Insights

10 • Autonomous Port Logistics System
Graph · Edge List
Kruskal · Dijkstra · Knapsack

Aarunya’s port needs to move containers efficiently between berths, yards and nearby warehouses using automated vehicles. The yard layout and road network are represented as weighted graphs where Dijkstra finds fast, fuel‑efficient routes for each vehicle. Kruskal’s algorithm constructs a minimum‑spanning tree over candidate conveyor or track segments to keep infrastructure cost down, and knapsack‑style DP fills each vehicle with containers so that weight and priority constraints are respected while trips are used to near capacity.

Aligned SDGs: SDG 9 – Industry, Innovation and Infrastructure, SDG 8 – Decent Work and Economic Growth

Data Structures: vector<Edge>, Union–Find, adjacency list — Algorithms: Kruskal MST, Dijkstra, 0/1 Knapsack DP
Berths / Yards → Edge List → Kruskal + DSU → Logistics Backbone Requests → Knapsack DP → Vehicle Load Plan Backbone Graph → Dijkstra → Container Routes
